# Clean Plan v0.3.1 — Hoàn thiện repo `opencode2claude`

## 0. Mục tiêu

Hoàn thiện repo sau release `v0.3.0` bằng một cleanup pass duy nhất, tập trung vào:

1. Xóa lệch giữa code thật và docs.
2. Chấm dứt dual control plane giữa `start.sh/stop.sh` và CLI supervisor.
3. Fix các lỗi High liên quan đến security, proxy retry, install path, docs drift.
4. Làm verification gates bắt lỗi thật, không còn pass giả.
5. Chuẩn bị repo đủ sạch để tag `v0.3.1`.

Không làm feature lớn mới trong plan này. Chỉ cleanup, fix correctness, harden security, và đồng bộ documentation.

---

## 1. Nguyên tắc bắt buộc

```text
1. CLI supervisor là control plane chính.
2. start.sh/stop.sh chỉ là compatibility wrapper, không được cạnh tranh với CLI.
3. WarmStandby 40004–40005 là protected infrastructure.
4. Không được stop/restart/purge/recreate/kill/detach WarmStandby.
5. Docs phải mô tả đúng code hiện tại, không mô tả future behavior.
6. Verification gates phải kiểm behavior thật, không pass bằng grep hình thức.
7. Không disable failing gates để pass.
8. Không tag v0.3.1 nếu còn High issue.
```

## A3. Convert `start.sh` thành compatibility wrapper

### Files

```text
start.sh
README.md
CLAUDE.md
docs/migration-from-start-sh.md
```

### Vấn đề

`start.sh` vẫn:

```text
- bắt buộc OpenCode CLI
- start opencode serve
- dùng .bridge.pids
- compile release binary
- spin Docker proxy pool
- nohup binary trực tiếp
```

Điều này cạnh tranh với CLI supervisor.

### Fix

`start.sh` mới chỉ làm:

```text
1. Check opencode2claude binary exists.
2. Optionally bootstrap Docker proxy pool nếu user bật legacy mode.
3. Gọi opencode2claude start.
4. Nếu sourced, export:
   ANTHROPIC_API_KEY
   ANTHROPIC_BASE_URL
   OPENCODE_MODEL
5. In rõ:
   "start.sh is a legacy compatibility wrapper. Prefer opencode2claude start."
```

Không còn:

```text
- opencode serve
- .bridge.pids
- kill rogue process
- nohup ./target/release/opencode2claude
```

### Acceptance gates

```bash
grep -q "opencode serve" start.sh && exit 1 || true
grep -q ".bridge.pids" start.sh && exit 1 || true
grep -q "opencode2claude start" start.sh
```

## A4. Fix `stop.sh` WarmStandby protection

### Files

```text
stop.sh
scripts/phases/phase-4-proxy-cli.sh
scripts/phases/phase-7-docs-migration.sh
docs/proxy-pool.md
docs/migration-from-start-sh.md
```

### Vấn đề

`stop.sh` đang stop/purge toàn bộ `opencode-warp-*`, có thể chạm `opencode-warp-4` và `opencode-warp-5`, tương ứng 40004–40005.

### Fix

Policy mới:

```text
stop.sh:
  default:
    - stop bridge only
    - không chạm proxy containers

stop.sh --stop-proxy:
    - stop only primary containers:
      opencode-warp-1
      opencode-warp-2
      opencode-warp-3
    - skip:
      opencode-warp-4
      opencode-warp-5

stop.sh --purge:
    - remove only primary containers:
      opencode-warp-1
      opencode-warp-2
      opencode-warp-3
    - skip warm-standby
```

Output bắt buộc:

```text
Skipping protected warm-standby proxies: 40004, 40005
```

### Acceptance gates

```bash
grep -q "opencode-warp-4" stop.sh && exit 1 || true
grep -q "opencode-warp-5" stop.sh && exit 1 || true
grep -q "Skipping protected warm-standby" stop.sh
./scripts/verify.sh phase-7 --profile ci
```

Gate Phase 7 phải fail nếu `stop.sh` có pattern stop/rm all `opencode-warp-*`.

# Phase B — Runtime / Proxy correctness

## B1. Fix `get_client_excluding()`

### Files

```text
src/proxy_pool.rs
src/opencode/forward.rs
```

### Vấn đề

`get_client_excluding(api_key, exclude_idx)` hiện không exclude thật.

### Fix

Implement:

```rust
select_proxy_for_key_excluding(routing_key, excluded_idx)
```

Behavior:

```text
1. Nếu assigned primary != excluded_idx và healthy → dùng assigned primary.
2. Nếu assigned primary == excluded_idx → không chọn lại proxy đó trong cùng retry.
3. Prefer warm-standby if assigned primary excluded/unhealthy.
4. Fallback degraded không được chọn excluded_idx trừ khi không còn lựa chọn nào.
```

### Tests

```rust

test_get_client_excluding_does_not_return_excluded_idx
```
